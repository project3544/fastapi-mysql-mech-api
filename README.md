## 📁 Проект в портфолио: REST API для Mech Pilot Service
Разработал современную бэкенд-систему на FastAPI для управления базой данных игровых персонажей (пилотов мехов). Приложение полностью готово к интеграции с фронтендом и мобильными клиентами.

### Что реализовано под капотом (архитектура):
1. **Строгая валидация (Pydantic V2):** Настроил модели `UserCreate` и `UserResponse` с использованием синтаксиса `ConfigDict(from_attributes=True)`. Сервер автоматически блокирует некорректные данные от фронтенда, защищая MySQL.
2. **Управление сессиями без утечек памяти:** Реализовал функцию `get_db` через `Depends`. Соединения с MySQL принудительно закрываются (`db.close()`) в блоке `finally` после каждого запроса, что освобождает оперативную память сервера.
3. **REST-роуты с автоматической документацией Swagger:**
   * `GET /pilots` — выгрузка списка персонажей из базы в виде чистого JSON-массива.
   * `GET /register/random` — генерация случайного пилота с автоматическим `db.refresh()` для получения ID.
   * `POST /register` — прием защищенного JSON-пакета от фронтендеров с использованием оператора распаковки `**user_data.model_dump()`.
   * `DELETE /pilots/{pilot_id}` — безопасное удаление пилота из базы по его ID с валидацией ошибок (404 Not Found).
4. **Отказоустойчивость транзакций:** Все операции записи и удаления обернуты в блоки `try-except`. При любом системном сбое срабатывает защитная команда `db.rollback()`, предотвращая повреждение и блокировку таблиц.
